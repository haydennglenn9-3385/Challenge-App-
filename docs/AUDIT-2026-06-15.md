## Architecture Notes for Future Development

- **DB access pattern**: CLAUDE.md documents `lib/storage.ts` as the central layer, but the app doesn't follow this. New features should use server actions (see `dailyCheckin.ts`, `validateJoinCode.ts`) — they run on the server, have access to the service-role key, and can verify auth before acting.
- **Admin operations**: Always use a server action that calls `supabaseAdmin` after verifying `role === 'admin'` from the session. See `updateMemberStats.ts` and `app/api/admin/update-user/route.ts` as the pattern.
- **Never return `join_code` to the client** — validate codes in `validateJoinCode.ts`.
- **Team membership**: Two tables (`team_members` + `challenge_members.team_id`) must stay in sync. Prefer `challenge_members.team_id` as the source of truth; `team_members` is legacy but still used by the messages page for team chat access.
